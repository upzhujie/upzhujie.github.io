http协议

发表于 2020-06-24 | 分类于 计算机网络

http协议 - 超文本传输协议， 是基于TCP/IP的应用层协议，它不涉及数据包传输，主要规定了客户端和服务器之间的通信格式

基本性质：

  • 简单：设计简单，报文容易阅读

  • 可拓展： 客户端和服务端可以拓展http headers，拓展功能

  • 无状态，有会话的： 两个http请求是不相关的，但是可以通过cookie来建立有状态的会话（本质无状态）

http是基于TCP协议的，因为TCP协议是可靠的，可以通过他的超时重传以及报文校验等等来保证数据的可靠

通过http能做什么

  • 通过Cache-Control控制缓存
  • 开放同源策略，Access-Control-Allow-Origin，使得跨域请求
  • 通过cookie建立一个带有状态的会话

关于http/1.0

在http1.0中，默认会为每一对单独的http请求都打开一个单独的TCP链接，例如在一个客户端中，同时请求了一个域名下的不同资源，他会为每一次请求都建立一个单独的TCP链接，因此这种方式是比较低效的

Q： 那么如果针对统一域名下的请求，都共享一个TCP链接，是否会提升效率呢？ - 会

关于http/1.1

在http1.1中，引入了持久链接，即TCP链接默认不关闭，可以被多个请求复用，即Connection: keep-alive(默认)，在我们建立第一次TCP链接后，TCP链接在第一次请求完成后并不会马上销毁，若有第二个对于同一域名的请求，则会复用该链接，否则，进入休眠状态，若在一定时间内发现客户端和服务端并没有请求了，则可以主动地关闭链接。

浏览器对于同一域名的请求，可以并发建立多个TCP链接，但一般数量会控制在4-8个之间

如果浏览器不对并发请求数量进行限制，会给服务器造成很大的压力，一时间会有大量请求同时发往服务端，造成堵塞

对于页面的优化方向： 减少资源请求

关于http/2

  • 帧： HTTP/2 数据通信的最小单位消息

  • 流：由帧组成数据流

多路复用

一个TCP链接中可以存在多个流，换句话说也就是可以发送多个请求，可以把每个请求当作一个流

同域名下所有通信都在单个连接上完成，该连接可以承载任意数量的双向数据流。每个数据流都以消息的形式发送，而消息又由一个或多个帧组成。多个帧之间可以乱序发送，根据帧首部的流标识可以重新组装。

结论

HTTP 报文是使用 HTTP 的关键；它们的结构简单，并且具有高可扩展性。HTTP/2 帧机制是在 HTTP/1.x 语法和底层传输协议之间增加了一个新的中间层（大概是从数据到帧和流的转化层），而没有从根本上修改它，即它是建立在经过验证的机制之上
