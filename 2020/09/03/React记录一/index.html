发表于 2020-09-03 | 分类于 React

记录一下React学习过程中遇到的问题以及思考

  • 关于React的视图更新

    在vue中，对于视图更新，我们只需要简单的设置我们的data数据，即能自动达到更新视图的效果，例如

    通过设置data中的数据，我们无需其他操作，便能达到视图的更新，并且若将数据通过props传输给子组件时，子组件便能自动的跟随父组件的视图而更新

    在react中，需要通过setState去设置对应的数据来达到更新视图的目的，然而在父子组件的更新中，可能会遇到问题，例如

    若以vue的思想去理解react的数据更新，则此时，父子组件都会保持同步更新，但实际的结果却是子组件不会实时更新，父组件会。

    大致可以推测，react的数据更新，需要依赖开发者去手动设置需要更新的视图，react内部保证最小颗粒地更新，若想要达到父子组件的实时更新，则需要开发者手动在子组件中设置相应的操作去达到目的，例如

    或者说，在子组件中直接使用this.props.sec去使用数据，由于父子组件对象的引用，在父组件中更改对应对象的数据时，便能实时反馈给子组件

  • 关于react中父子组件传值的思考

    在vue中，最简单的父子传值，通过props讲父组件数据传递给子组件，在子组件中通过emit触发父组件对应方法，达到父子通信的效果。

    在react中，也可以实现大致的耦合，在父组件中函数通过props传递给子组件，在子组件中便可使用该函数

    例如

    在实际的使用场景中，例如有很多子组件，依赖同一个父组件数据，那么此时，我们需要在子组件中，将变更数据的方法在父组件中执行，达到所有组件同时同步刷新视图的目的

  • 使用大致体验与感受

    在vue中，作者给我们做好了非常多的操作，让我们能够更专注于数据的变化，而比较少的去关注视图的更新情况，哪里需要更新，我们很少去关心，这样其实也带来了一些弊端，例如diff的过程其实对性能的消耗是不可忽视的，而react，则需要我们去关注哪里需要我们更新的视图，关于react的视图更新的效率，后续将会深入的去探究。

    在vue3中，vnode引入了一个patchFlag的概念，快速拿到vnode中动态节点，避免多余节点的重复diff，能够大幅度提高更新视图的效率，待续…
